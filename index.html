SDLC

Software Development Life Cycle (SDLC) is a structured process used by software development teams to design, develop, test and deploy high-quality software systems. It consists of a series of phases,

Requirement Analysis: In this phase, stakeholders collaborate to gather and analyze requirements for the software system. They understand the needs of the end-users, define functional and non-functional requirements and document these requirements properly.

Design: During the design phase, the system architecture, data structures, interfaces and algorithms are designed based on the gathered requirements. In this phase, we come up with detailed technical specifications, architecture diagrams and design documents and these will be used as blueprints in the upcoming phases.

Implementation: In the Implementation phase, developers write code based on the specifications and designs created in the previous phases. Here, they translate the design documents into executable software components, following coding standards and best practices and integrating various modules to build the complete system.

Testing: In the testing phase we verify and validate the software to ensure that it meets the specified requirements and functions correctly. Different types of testing, such as unit testing, integration testing, systems testing and acceptance testing are performed to identify and fix defects and ensure software quality.

Deployment: Once the software has been thoroughly tested and approved, it is deployed to production. This involves configuring the software environment, installing the application on target systems and ensuring that it functions as expected in the production environment.

Maintenance: After deployment, the software enters the maintenance phase, where it is actively supported, monitored and updated to address issues, add new features or accommodate changes in user requirements. Maintenance activities may include bug fixes, performance optimization, security updates and enhancements to existing functionality.

Angular JS

It is a JavaScript framework and is used to build user interfaces.

It is developed by Google (In 2010) and it follows the MVC (Model-View-Controller) architecture for structuring the applications.

It provides built-in support for features like routing, form validation and dependency injection.

It uses HTML as a template language with additional syntax and directives.

It uses two-way data binding, meaning changes in the UI elements are automatically reflected in the underlying data model and vice versa.

React JS

It is a JavaScript framework and is used for building user interfaces.

It is developed by Facebook(In 2013), and it follows a component-based architecture, where the UI is broken down into reusable components.

It uses a virtual DOM (Document Object Model) to efficiently update the UI. It calculates the difference between the previous and current states of the virtual DOM and only applies necessary changes to the real DOM, resulting in the improved performance.

It uses JSX as its syntax extension for JavaScript, allowing us to write HTML-like code in JavaScript files.

It encourages unidirectional data flow, meaning data flows in a single direction through the component hierarchy.

Next JS

Next.js is a popular framework for building React applications and was initially released in 2016.

It allows server-side rendering, which means that pages are rendered on the server before being sent to the client’s browser. This can improve performance and SEO by delivering fully rendered HTML pages to the client.

It provides a built-in routing system that allows us to define routes for the application’s pages.

JavaScript

JavaScript is a high-level, interpreted programming language primarily used for building dynamic and interactive web applications.

JavaScript is primarily used as a client-side scripting language, meaning it runs on the user's web browser rather than on the server. This allows developers to create dynamic and interactive user interfaces that respond to user actions in real-time without needing to reload the entire web page.

JavaScript is commonly used for web development to add interactivity, validate user input, manipulate the DOM (Document Object Model), handle events, and perform asynchronous operations like fetching data from servers (AJAX).

TypeScript

TypeScript introduces static typing to JavaScript, which allows us to define types for variables, function parameters, return values etc. This helps in identifying type-related errors during development and it helps in documenting the code better and it's now more readable.

It features type inference, which means that the compiler can automatically infer the types of variables based on their usage. With this, we don’t have to specifically mention the type, so the development process gets a little faster.

It offers better tooling support, including features like code navigation, intelligent code completion and refactoring tools. So, this improves our productivity and will help us maintain large codespaces better.

Difference between JavaScript and TypeScript

TypeScript allows developers to define types for variables, function parameters, return values, and other entities. JavaScript, on the other hand, is dynamically typed, meaning types are determined at runtime rather than at compile time.

In TypeScript, developers can explicitly annotate variables, function parameters, and other entities with types. JavaScript does not support type annotations.

TypeScript features type inference, where the compiler can automatically deduce the types of variables based on their usage. JavaScript does not have type inference.

TypeScript performs static type checking at compile time, catching type-related errors before the code is executed. JavaScript, being dynamically typed, does not have this level of compile-time checking and may only catch type-related errors at runtime.

React Redux

React Redux is a library used for state management in React applications.

It provides a centralized store to manage the application's state, enabling components to access and update the state without having to pass data through props.

The connect function allows components to connect to the Redux store and access the state or dispatch actions, improving performance by selectively subscribing to relevant parts of the state.

Additionally, React Redux supports middleware for intercepting and modifying actions, and it integrates with the Redux DevTools extension for powerful debugging capabilities.

MongoDB Database

MongoDB is a document-oriented NoSQL database known for its flexibility, scalability, and ease of use.

It stores data in flexible JSON-like documents, making it suitable for handling unstructured and semi-structured data.

It supports dynamic schemas, horizontal scalability through sharding, and built-in replication for high availability.

It is commonly used in modern web applications, mobile app backends, real-time analytics, and content management systems. It is well-suited for applications requiring flexible data models, rapid development cycles, and scalability.

Node JS

It is an open-source, cross-platform JavaScript runtime environment that allows developers to run JavaScript code server-side.

It uses a single threaded event loop model to handle concurrent requests. This allows it to handle multiple connections without the need for creating separate threads for each connection.

Npm is the default package manager for Node.js. Developers can easily install, manage and execute associated callback functions, making Node.js highly scalable and performant.

It is commonly used for server-side development, including building web servers, APIs(Application Programming Interfaces) and real-time applications like chat applications and multiplayer-games

It is cross platform, meaning it can run on various operating systems including Windows, MacOS and Linux, making it versatile and adaptable for different environments.

Express JS

It's like the framework you use to build the backend part of our website.

It provides all the tools and structure you need to create a solid backend. With Express, you can easily handle things like receiving requests from users' browsers, sending back responses, managing routes, and adding extra features with middleware.

It's popular because it's simple and flexible, it lets us build powerful server-side applications without a lot of hassle.

Difference between Node.js and Express.js

Express.js is built on top of Node.js, which means it's a framework specifically designed to work with Node.js.

Think of Node.js as the engine that runs your server-side code, while Express.js provides the structure and tools to build your server-side applications on top of Node.js.

So, when you use Express.js, you're essentially using Node.js under the hood to power your server and handle requests and responses.

Express.js simplifies the process of building server-side applications with Node.js by providing a more user-friendly and streamlined framework.

JWT Tokens

JSON Web Tokens are a secure way of transmitting information between parties in a web application.

Essentially, they're like digital passports that contain user-related information. When a user logs in, the server generates a JWT token, which is then sent to the client.

This token is typically stored on the client side, either in local storage or as a cookie.

Whenever the user makes a request to the server, they include this token in the request. The server can then verify the token to ensure that the request is coming from an authenticated user and has the necessary permissions to access the requested resources.

JWT tokens are widely used for authentication and authorization in web applications due to their security and efficiency.

DOM

Document Object Model, or DOM, is essentially the way web browsers represent web pages internally.

It creates a structured representation of the HTML elements on a web page, forming a tree-like structure where each element, like paragraphs, headings, or buttons, is a node in the tree.

This allows developers to use JavaScript to interact with and manipulate the content, structure, and style of a web page dynamically.

For example, you can use the DOM to access elements on a web page, change their content or appearance, respond to user interactions like clicks or keystrokes, and even create or remove elements altogether.

Single Page Applications

Single Page Applications are a type of web application where the entire application runs on a single web page.

Unlike traditional multi-page applications, which require loading a new page from the server every time the user interacts with the application, SPAs load all necessary code when the user first accesses the application.

From there, interactions with the application are handled dynamically on the client side, typically using JavaScript frameworks like React, Angular, or Backbone.js.

This approach provides a smoother and more seamless user experience, as content updates happen quickly without the need for full page reloads.

Backbone JS

Backbone.js is a lightweight JavaScript framework that's often used to add structure to web applications.

It provides a set of tools for organizing and managing the data, views, and user interactions within an application.

Backbone.js is particularly useful for building single-page applications (SPAs) where a lot of the application logic is handled on the client side.

It's known for its simplicity and flexibility, making it a popular choice among developers who want to build interactive web applications without the overhead of larger frameworks.

RESTful APIs

RESTful APIs are a set of architectural principles for designing networked applications.

In simpler terms, they're a way of building web services that are easy to understand and use. RESTful APIs are based on the idea of treating resources as unique identifiers that can be manipulated using a standardized set of operations.

These operations include things like creating, reading, updating, and deleting resources, which map to the HTTP methods of POST, GET, PUT, and DELETE respectively.

RESTful APIs use standard HTTP protocols and status codes to communicate between clients and servers, making them highly interoperable and easy to integrate with existing web technologies.

GraphQL

GraphQL is a query language and runtime for APIs that enables clients to request only the data they need.

Unlike traditional REST APIs, where clients receive fixed data structures, GraphQL allows clients to specify the structure of the data they require in a single query, reducing over-fetching and under-fetching of data.

GraphQL APIs are commonly used in modern web and mobile applications to improve efficiency and flexibility in data fetching.

GraphQL Subscriptions

GraphQL subscriptions are a feature of GraphQL that allows clients to receive real-time updates from the server.

In simpler terms, it's like setting up a live connection between the client and server, so the server can push updates to the client whenever certain events occur.

This is particularly useful for applications where users need to see real-time updates, like chat apps or live feeds.

Instead of the client having to constantly ask the server for updates, GraphQL subscriptions enable a more efficient and responsive way of handling real-time data.

It offers services like Amazon EC2 for virtual servers, Amazon S3 for object storage, Amazon RDS for managed relational databases, Amazon DynamoDB for NoSQL databases, Amazon Lambda for serverless computing and many others.

With AWS, organizations can easily deploy and manage applications, scale resources based on demand, improve flexibility and agility and enhance overall IT efficiency.

Setting up DevOps Pipelines with Jenkins

Setting up DevOps pipelines with Jenkins is a process of automating the continuous integration and continuous delivery (CI/CD) of software applications.

Jenkins is an open-source automation server that helps streamline the software development lifecycle by automating various tasks, such as building, testing, and deploying code changes.

Test-Driven Development

Test-Driven Development is a software development approach where tests are written before writing the actual code.

The development process typically follows three steps: write a failing test, write the minimum amount of code to pass the test, and then refactor the code while ensuring that all tests still pass.

TDD promotes a cycle of rapid feedback, encourages better design and helps ensure that code is thoroughly tested.

It often leads to cleaner, more maintainable codebases and reduces the likelihood of introducing bugs or regressions.

CI/CD

CI/CD is a set of practices used in software development to automate the process of integrating code changes into a shared repository (Continuous Integration) and deploying code changes to production environments (Continuous Deployment).

Continuous Integration involves frequently integrating code changes into a shared repository, running automated tests and detecting integration errors early in the development process.

Continuous Deployment automates the deployment of code changes to production environments after passing automated tests and quality checks, enabling faster and more reliable releases.

CI/CD pipelines typically include stages for building, testing, deploying and monitoring applications, ensuring a streamlined and automated development workflow.

Jasmine

Jasmine is a behavior-driven development (BDD) framework for testing JavaScript code.

It provides a clean and expressive syntax for writing tests, making it easy to define test suites and individual test cases.

It also comes with built-in matchers and spies for asserting and mocking behavior within tests.

Jest

Jest is a testing framework developed by Facebook.

It's designed to be fast, easy to set up, and highly customizable.

Jest is particularly popular in the React community and is often used alongside tools like React Testing Library or Enzyme for testing React components.

It comes with built-in features like snapshot testing, which allows us to capture and compare component output, as well as powerful mocking capabilities for isolating components and dependencies during testing.

Difference between Jasmine and Jest

Jest comes with built-in mocking capabilities, whereas Jasmine requires additional libraries like Sinon.js for mocking.

Jest supports snapshot testing out of the box, allowing developers to capture and compare component output. Jasmine does not have native support for snapshot testing.

Jest is known for its fast test execution and parallel test running capabilities, making it suitable for large codebases.

Less/Sass

Less and Sass are both CSS preprocessors, which means they are tools that extend the functionality of CSS.

Less and Sass introduce enhancements to the CSS syntax, such as variables, mixins, nesting, and inheritance. These features make CSS code more modular, maintainable, and easier to write.

Both of them allow developers to define variables to store values like colors, sizes, and fonts. This makes it simple to update these values across the stylesheet and ensures consistency throughout the design.

They support nesting of CSS selectors, allowing developers to write CSS in a more hierarchical and structured manner. This nesting mirrors the structure of HTML markup, making the styles easier to understand and maintain.

Differences between Less&Sass

Less uses a CSS-like syntax with minimal changes, while Sass has its own syntax, which is more expressive and powerful.

Sass has a more extensive feature set, including powerful features like control directives (@if, @for, @each, @while), placeholders (%placeholder), and modules (@use and @import).

Less has a simpler feature set compared to Sass, with fewer built-in functions and features.

AJAX

AJAX, which stands for Asynchronous JavaScript and XML, is a web development technique used to create interactive and dynamic web applications.

This enables web pages to send and receive data from a server asynchronously, meaning it can perform tasks in the background without interfering with the user's experience on the webpage.

With AJAX, web pages can update specific parts of the page dynamically without requiring a full page reload. This allows for a smoother and more responsive user experience.

Many JavaScript frameworks and libraries, such as jQuery, AngularJS, and React, provide built-in support for AJAX

AJAX is commonly used for features such as form submissions, live search suggestions, auto-complete fields, and updating content dynamically without reloading the entire page.

jQuery is a fast, small, and feature-rich JavaScript library designed to simplify event handling, animating, and Ajax interactions for rapid web development.

This provides a simple and concise syntax for selecting and manipulating HTML elements in the DOM. We can easily modify the DOM tree, manipulate element attributes, content, and styles, and also handle events using jQuery methods and selectors.

SQL Server

As an RDBMS, SQL Server organizes data into tables consisting of rows and columns, with each table representing a different entity. It supports the SQL for querying and manipulating data, allowing us to perform tasks such as retrieving, updating, inserting, and deleting data from the database.

No SQL

A NoSQL database is a type of database system that differs from traditional SQL databases in how it stores and manages data. While SQL databases use a structured, tabular format with predefined schemas, NoSQL databases employ a more flexible and dynamic approach to data storage. They are designed to handle large volumes of unstructured or semi-structured data, such as documents, graphs, key-value pairs, or wide-column stores, which may not fit neatly into the rows and columns of a relational database. NoSQL databases are often used in scenarios where scalability, flexibility, and performance are paramount, such as web applications, big data analytics, and real-time data processing.

Selenium supports multiple programming languages such as Java, Python, C#, and JavaScript, allowing users to write automated tests in their preferred language. It is widely used in software testing to automate repetitive tasks, perform regression testing, and ensure the functionality and stability of web applications across different browsers and platforms.
